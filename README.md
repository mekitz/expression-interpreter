# Интерпретатор арифметико-логических выражений

Интерпретатор арифметико-логических выражений, построенный на основе **метода Дейкстры** (сортировочная станция) и **стекового арифметико-логического устройства (САЛУ)**.  
Проект выполнен в рамках учебного задания по дисциплине «Информатика».

## Сигнатура

Выражения строятся по следующей сигнатуре (приоритеты и ассоциативность):

| Оператор | Тип | Ассоциативность |
|----------|-----|-----------------|
| `^` | бинарный | левоассоциативный (`→`) |
| `~` | унарный | левоассоциативный (`→`) |
| `*` | бинарный | правоассоциативный (`←`) |
| `/` | бинарный | левоассоциативный (`→`) |
| `&` | бинарный | левоассоциативный (`→`) |
| `+(1)` | унарный | левоассоциативный (`→`) |
| `-(1)` | унарный | левоассоциативный (`→`) |
| `+(2)` | бинарный | левоассоциативный (`→`) |
| `-(2)` | бинарный | левоассоциативный (`→`) |
| `<` | бинарный | правоассоциативный (`←`) |
| `>` | бинарный | правоассоциативный (`←`) |
| `=` | бинарный | правоассоциативный (`←`) |
| `#` | бинарный | правоассоциативный (`←`) |
| `!` | бинарный | правоассоциативный (`←`) |

### Обозначения:
- `(1)` — унарная операция
- `(2)` — бинарная операция
- `→` — левоассоциативная (выполняется слева направо)
- `←` — правоассоциативная (выполняется справа налево)

## Возможности

- **Арифметические операции:** `+`, `-`, `*`, `/`, `^` (возведение в степень)
- **Логические операции:** `&` (конъюнкция, И), `!` (дизъюнкция, ИЛИ), `~` (отрицание, НЕ)
- **Операции сравнения:** `<`, `>`, `=`, `#` (не равно)
- **Унарные операторы:** `+`, `-`, `~`
- **Функции:** `arctg(формула)`, `arcctg(формула)`
- **Приоритеты и ассоциативность** операций (согласно сигнатуре)
- **Вывод обратной польской записи (ОПЗ)** перед вычислением
- **Обработка ошибок:** деление на ноль, недостаток операндов, неизвестные операторы

## Описание основных функций

### Лексический анализ (`lexer.h` / `lexer.cpp`)

- `vector<Token> tokenize(const string& expr)`
  Разбивает строку выражения на последовательность токенов.
  Распознаёт числа, операторы, функции, скобки.
  Определяет, является ли оператор унарным или бинарным.

### Метод Дейкстры (`shunting_yard.h` / `shunting_yard.cpp`)

- `vector<Token> shuntingYard(const vector<Token>& tokens)`
  Преобразует выражение в обратную польскую запись (ОПЗ) с учётом приоритетов и ассоциативности операторов.
  Реализует алгоритм сортировочной станции.

- `int getPriority(const string& op, bool isUnary)`
  Возвращает приоритет оператора (от 5 — унарные, до 0 — дизъюнкция).

- `bool isRightAssociative(const string& op)`
  Определяет, является ли оператор правоассоциативным (`*`, `<`, `>`, `=`, `#`, `!`).

### Вычисление на САЛУ (`evaluator.h` / `evaluator.cpp`)

- `double evaluateRPN(const vector<Token>& rpn)`
  Вычисляет выражение в ОПЗ с помощью стека.
  Поддерживает унарные и бинарные операции, функции `arctg` и `arcctg`.

- `double applyOp(double a, double b, int opCode)`
  Выполняет бинарную операцию по коду: арифметическую, логическую или сравнение.

- `double applyUnaryOp(double a, int opCode)`
  Выполняет унарную операцию: смена знака или логическое отрицание.

### Точка входа (`main.cpp`)

- `main()`
  Запрашивает выражение, вызывает лексер, метод Дейкстры и САЛУ.
  Выводит ОПЗ и результат. Обрабатывает ошибки.
